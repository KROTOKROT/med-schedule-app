## 2. Бизнес-контекст

### Проблема
Многие пользователи забывают вовремя принимать лекарства, особенно при сложных схемах лечения (несколько препаратов, разные интервалы, курсы). Бумажные заметки и обычные будильники не обеспечивают структурированного контроля и истории приёма.

### Решение
Разрабатывается мобильное приложение, которое позволяет пользователю создавать расписание приёма лекарств, получать уведомления, отслеживать историю приёма и работать в офлайн-режиме с последующей синхронизацией с сервером.

---

## 3. Цели проекта

- Обеспечить удобное создание и управление расписанием приёма лекарств
- Реализовать систему уведомлений о приёме препаратов
- Обеспечить хранение истории приёма и синхронизацию с сервером
- Реализовать безопасную авторизацию пользователей (JWT)

---

## 4. Ключевые показатели успеха

- Пользователь может создать расписание менее чем за 1 минуту
- Уведомления доставляются с точностью не менее 95%
- Поддержка офлайн-режима без потери данных
- Время ответа API < 300 мс для основных запросов

---

## 5. Ключевые риски

- Ошибки синхронизации офлайн/онлайн данных  
  → Решение: локальная БД + стратегия конфликтов

- Пропуск уведомлений  
  → Решение: системные уведомления Android + резервный механизм

- Потеря данных при сбоях  
  → Решение: резервирование и транзакционная модель
