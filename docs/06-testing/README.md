# 07. Refactoring (Рефакторинг и улучшение качества кода)

## 1. Общая информация

На данном этапе выполнен рефакторинг серверной части системы MedSchedule с целью улучшения архитектуры, повышения читаемости кода и соответствия принципам чистой архитектуры.

---

## 2. Цели рефакторинга

- улучшение разделения ответственности между слоями PCMEF
- устранение дублирования логики
- повышение тестируемости кода
- приведение структуры к промышленному стилю

---

## 3. Применённые паттерны

### 3.1 Data Mapper

Используется для отделения доменных объектов от слоя доступа к данным.

Ранее:
- Entity напрямую использовались в сервисах

После рефакторинга:
- добавлены DTO
- преобразование происходит в отдельном слое mapper

---

### 3.2 Identity Map

Используется для предотвращения повторной загрузки одинаковых сущностей в рамках одной сессии.

Преимущества:
- уменьшение количества запросов к БД
- консистентность данных в памяти

---

### 3.3 Улучшение Service слоя

- разделена бизнес-логика на отдельные методы
- убраны “толстые” контроллеры
- логика перенесена в Mediator слой

---

## 4. Пример рефакторинга

### До:

Service напрямую работал с Entity без разделения:

public Medication create(Medication med) {
    return repository.save(med);
}

---

### После:

Добавлен DTO и Mapper слой:

public MedicationDto create(MedicationDto dto) {
    Medication med = mapper.toEntity(dto);
    Medication saved = repository.save(med);
    return mapper.toDto(saved);
}

---

## 5. Улучшения архитектуры

- контроллеры стали “тонкими”
- бизнес-логика централизована в Service
- добавлен слой преобразования данных (DTO/Mapper)
- улучшена читаемость PCMEF структуры

---

## 6. Статический анализ

После рефакторинга:
- устранены критические замечания
- снижена циклическая сложность
- улучшена структура пакетов

---

## 7. Результаты

- код стал более модульным
- увеличилась тестируемость
- снижено количество зависимостей между слоями
- улучшена поддерживаемость проекта

---

## 8. Вывод

Рефакторинг привёл систему к более чистой архитектуре, соответствующей принципам PCMEF и промышленным стандартам разработки.
