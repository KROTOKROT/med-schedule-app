# 08. UI (Пользовательский интерфейс)

## 1. Общая информация

На данном этапе описан пользовательский интерфейс мобильного приложения MedSchedule.

UI реализует взаимодействие пользователя с системой управления расписанием приёма лекарств и обеспечивает выполнение основных сценариев: добавление лекарств, просмотр расписания, управление уведомлениями и настройками приложения.

---

## 2. Общая структура интерфейса

Мобильное приложение содержит **5 основных экранов**, что соответствует требованиям курсового проекта:

- Главный экран (Home)
- Список лекарств (Medication List)
- Добавление лекарства (Add Medication)
- Детали лекарства (Medication Details)
- Настройки (Settings)

---

## 3. Описание экранов

### 3.1 Главный экран (Home)

Функции:
- отображение ближайших приёмов лекарств
- быстрый доступ к расписанию
- статус уведомлений

Элементы UI:
- список ближайших приёмов
- карточки лекарств
- кнопка перехода к добавлению

---

### 3.2 Экран списка лекарств

Функции:
- отображение всех добавленных лекарств
- переход к деталям лекарства
- удаление лекарства

Элементы UI:
- RecyclerView / LazyColumn
- карточки лекарств
- кнопка удаления

---

### 3.3 Экран добавления лекарства

Функции:
- добавление нового лекарства в систему
- настройка дозировки и времени приёма

Элементы UI:
- поля ввода (название, дозировка)
- выбор времени (TimePicker)
- кнопка сохранения

---

### 3.4 Экран деталей лекарства

Функции:
- просмотр полной информации о лекарстве
- редактирование данных

Элементы UI:
- название лекарства
- расписание приёма
- кнопка редактирования

---

### 3.5 Экран настроек

Функции:
- управление уведомлениями
- настройки синхронизации
- информация о приложении

Элементы UI:
- toggle switch уведомлений
- кнопка синхронизации
- информация о версии

---

## 4. Навигация

Навигация реализована через:

- Bottom Navigation Bar
- Navigation Component (Android) / Navigator (Flutter)

Переходы между экранами:
Home → List → Details → Add → Settings

---

## 5. Архитектура UI слоя

UI реализован в рамках PCMEF:

- Presentation: Activity / Fragment / Compose UI
- Control: ViewModel
- Mediator: Repository
- Entity: модели данных
- Foundation: API + Room

---

## 6. Состояния интерфейса

Каждый экран поддерживает 3 состояния:

- Loading (загрузка данных)
- Success (данные загружены)
- Error (ошибка загрузки)

---

## 7. Ключевые UI технологии

- Jetpack Compose / XML
- Material Design 3
- ViewModel + StateFlow
- Navigation Component
- Retrofit (связь с API)
- Room (локальное хранение)

---

## 8. Соответствие требованиям

UI удовлетворяет требованиям курсового проекта:

- минимум 5 экранов ✔
- навигация между экранами ✔
- CRUD операции ✔
- обработка состояний ✔
- интеграция с backend API ✔
- поддержка оффлайн режима (через кеш) ✔

---

## 9. Вывод

Пользовательский интерфейс реализует все ключевые сценарии взаимодействия с системой и обеспечивает удобную работу с расписанием приёма лекарств как в онлайн, так и в оффлайн режиме.
