# 12. Final Report (Пояснительная записка)

## 1. Общая информация

Курсовой проект: **MedSchedule — мобильное приложение для управления расписанием приёма лекарств**

Траектория: Mobile (Android + Spring Boot backend)  
Архитектура: PCMEF  
Стек: Java 17, Spring Boot, PostgreSQL, Flutter/Android, JWT, REST API

---

## 2. Актуальность проекта

В современном мире большое количество людей принимает лекарственные препараты по сложным схемам. Ошибки в приёме могут привести к снижению эффективности лечения.

Существующие решения часто:
- не поддерживают оффлайн режим
- имеют ограниченную кастомизацию расписаний
- не обеспечивают гибкую систему уведомлений

---

## 3. Цель и задачи

### Цель:
Разработка мобильного приложения для управления расписанием приёма лекарств с серверной поддержкой.

### Задачи:
- анализ предметной области
- проектирование требований
- разработка архитектуры PCMEF
- реализация backend и mobile client
- реализация REST API
- внедрение JWT безопасности
- реализация оффлайн режима
- тестирование системы
- подготовка документации

---

## 4. Аналитическая часть

Были выполнены:
- бизнес-анализ предметной области
- построение модели бизнес-процессов
- определение ключевых сущностей:
  - пользователь
  - лекарство
  - расписание
  - уведомление

---

## 5. Проектирование системы

### 5.1 Архитектура

Система построена на паттерне PCMEF:

- Presentation — мобильное приложение
- Control — REST контроллеры
- Mediator — бизнес-логика
- Entity — доменные сущности
- Foundation — доступ к данным

---

### 5.2 База данных

Используется PostgreSQL.

Основные сущности:
- users
- medications
- schedules
- notifications

---

## 6. Реализация

### Backend:
- Spring Boot REST API
- JWT аутентификация
- CRUD операции
- Swagger документация

### Mobile:
- 5+ экранов
- навигация между экранами
- локальный кеш (Room/SQLite)
- уведомления

---

## 7. API

Реализовано 8+ эндпоинтов:
- /auth/login
- /auth/register
- /users/me
- /medications
- /schedule
- /notifications

Документация доступна через Swagger UI.
